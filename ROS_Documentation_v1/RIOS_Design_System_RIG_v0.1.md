# RIOS Design System (RIG)

## Relationship Interface Guidelines

**Version:** A0.1

## Philosophy

RIOS is not a CRM. It is a Relationship Intelligence Operating System.

Every visual decision should make the owner feel like an intelligent
Chief Relationship Officer is sitting beside them.

Core principles: - Relationships first, channels second. - Calm
interfaces over busy dashboards. - Information hierarchy over
decoration. - AI is a teammate, not a popup. - Consistency over
creativity. - Density without clutter.

## Visual Language

Inspired by: - Apple - Linear - Vercel - Notion - Arc Browser - ChatGPT

Avoid: - Salesforce - HubSpot - Traditional CRM dashboards

## Layout

Three-column layout:

1.  Left Navigation
2.  Center Work Queue
3.  Right Relationship Advisor

This layout remains the default across RIOS.

## Relationship Card Doctrine

Relationship cards are mission cards, not spreadsheet rows.

Each card answers:

-   WHO?
-   WHY TODAY?
-   COMMERCIAL OBJECTIVE?
-   CURRENT STAGE?
-   NEXT ACTION?
-   CONFIDENCE?

## Relationship Advisor

The right panel is the Chief Relationship Officer.

Responsibilities:

-   Explain reasoning
-   Coach communication
-   Challenge ideas
-   Suggest better strategy
-   Draft only after agreement

Conversation first. Generation second.

## Stitch Workflow

Humans design.

Documentation is updated.

Stitch reproduces the design faithfully.

Backend implements behaviour.

Do not let Stitch invent UX.

## Current Architectural Decision

The original Command Center mockup is the canonical design reference.

Stitch-generated layouts are implementation scaffolds only.

The existing HTML structure is reusable, but the design language follows
this document.

## Long-term Goal

Build a timeless design system so every future screen is assembled from
reusable components instead of redesigned from scratch.
